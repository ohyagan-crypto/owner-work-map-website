01

不是展示頁，是任務入口

主人直接在 Telegram 下指令，bot 依任務類型選擇工具與技能，不需要另外切換一堆平台。

02

結果要能拿來用

網站給公開網址，圖片與文件回傳成品，影片與音訊完成後回傳檔案，狀態問題則回答實際進度。

03

卡住會說清楚

遇到登入、驗證、付款、權限、工具無回應或同一步驟重複失敗，會停下回報原因與下一步。

Core Abilities

這隻 tgbot 可以接的主要任務

站

網站建置與修復

建立 GitHub Pages 網站、修復報名頁、部署更新、檢查公開頁面與前後端連線。

圖

圖片與海報任務

依照素材與主人指令進行圖像生成、修圖、風格延伸與成品回傳。

影

影片與剪輯流程

分析對標影片、整理字幕節奏、協助剪映流程與影片規格交付。

資

NotebookLM 與資料整理

整理來源、建立摘要、產出音訊或影片摘要，下載後依格式回傳。

監

自動檢查與修復

固定檢查指定網站健康狀態，能修的問題先修復，再重新驗證。

交

技能交接與文件

整理技能包、安裝說明、流程文件與交付檔案，避免混入密鑰或私人資料。

How It Works

從一句 Telegram 指令，到可交付成果

1

主人下指令

支援文字、網址、圖片、影片、文件與回覆引用素材。

2

判斷任務與素材

先看最新訊息，不把舊圖片、舊任務或舊記憶誤當成新需求。

3

執行與驗證

依任務操作本機工具、瀏覽器、網站部署或檔案轉換。

4

回傳成果

完成後回傳網址、檔案、驗證結果，或說明精確卡點。

Monitoring

網站與長任務不讓主人空等

針對已設定的網站任務，會固定檢查可用性。若遇到後端入口斷線、名單讀不到、前台 API 指向錯誤或備援資料異常，會先嘗試修復，再重新檢查結果。

每小時檢查

定時確認網站與關鍵流程。

錯誤自修復

可修復問題先處理，不只回報故障。

卡點即回報

登入、驗證、權限或付款不自行越過。
