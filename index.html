      記憶、技能、工作流，統一下載與套用

      之後你把更新檔交給我，我會先排除敏感資料、加密打包、更新這裡的清單，再部署到公開站。使用者輸入密碼後，可以下載更新包，或直接複製升級指令貼給自己的 TGBOT 自動完成更新。

      安全邊界 GitHub Pages 是公開靜態站，管理者密碼只用來解鎖瀏覽器端的選檔檢查；正式上架仍由管理端檢查後部署。更新包本身使用 AES-256-GCM 加密，不要把金鑰、Cookie、瀏覽器資料或原始對話紀錄放進更新檔。

      傳給 TGBOT

      複製升級指令，貼上就能更新

      在上方輸入 bsmf 解鎖後，點更新包旁的「複製升級／安裝指令」。Codex 技能包會先乾跑驗證；完整 TGBOT ZIP 則會下載、解壓並執行保留設定的安裝器。

      1輸入 bsmf 解鎖更新包。
      2按「複製升級指令」，不用自己修改路徑。
      3回到 Telegram，把複製好的升級指令傳給自己的 TGBOT。
      4TGBOT 會先乾跑檢查，通過後正式套用、重啟並回報驗證結果。
      進階：手動使用套用工具

      需要自行操作時，可下載套用工具。請先加上 --dry-run 檢查，驗證通過後再移除 --dry-run 正式更新。

      你交給我的更新流程 1. 你把記憶、技能、工作流更新檔傳給我。 2. 我檢查並排除金鑰、Cookie、瀏覽器資料與原始紀錄。 3. 我把合格的 .bsmf 放進更新清單並公開部署。 4. 使用者輸入 bsmf，複製升級指令貼給自己的 TGBOT 完成更新。 下載管理端上架工具：收到新包後先檢查，再更新公開清單。
